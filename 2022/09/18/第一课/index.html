第一课

2022/9/18 hexo

第一次建立一个属于自己的博客，参考了UP主:CodeSheep的教程。UP主讲的非常详细，照着他的步骤做，即使操作系统并不相同，但也没有出现大的问题。这篇博文会详细记录我博客的建立以及摸索过程，同时也要感谢我的朋友大鲨子对我学习过程中提供的帮助。

第一节-博客的建立

赛前准备

建立博客首先需要下载安装 nodejs ，以及 git 。安装无脑下一步即可。

然后使用管理员模式打开一个终端（包括但不限于cmd，powershell，以及类似于VScode这类带有终端功能的编辑器）

可以 win+r 输入 cmd 来打开 cmd 然后在任务栏右键点击 cmd 再在弹出的窗口右键点击命令提示符 以管理员身份运行 。

对于一些可以打开，但找不到程序位置的软件可以用这个笨方法来打开管理员模式。

在打开终端后可以输入以下命令来检验 node.js 、 npm 、 git 的安装成功与否

node -v
npm -v
git -v

若未出现类似于

xxxx : 无法将“xxxx”项识别为 cmdlet、函数、脚本文件或可运行程序的名称。请检查名称的拼写，如果包括路径，请确保路径正确，然后再试一次。所在位置 行:1 字符: 1

这样的提示，而是出现类似于 v16.17.0 8.15.0 git version 2.37.3.windows.1 这样的提示，则表示前置软件已安装成功。否则就按照以上步骤安装再重启。

赛前准备 2.0

检查完成后就可以输入以下命令来安装淘宝的 cnpm 管理器

npm install -g cnpm --registry=http://registry.npm.taobao.org

同样的可以使用以下命令来检查版本和安装情况

cnpm -v

随后还需要使用以下命令安装 hexo 框架

cnpm install -g hexo-cli

同样

hexo -v

在终端里进入你喜欢的一个文件夹 输入以下命令创建并进入此文件夹

mkdir blog
cd blog

其中blog可以换成任意你喜欢的文件夹名称，文件夹只是将整个工程划分一个场地装起来而已。

创建博客-只需要一条命令

确保你所在的文件夹是你想让博客存在的文件夹,且终端是以管理员模式打开的。

然后输入以下命令

hexo init

这样你的博客就创建好了。此处易出现证书问题引发的bug，详情以及解决办法请看附录部分。

第二节-检阅并编辑博客

检阅博客

目前来看，你的博客只是一个没有生气的工程文件，所以我们需要启动它。

hexo s

完成后会出现如下提示，且此时终端并不可以键入命令。

INFO  Validating config
INFO  Start processing
INFO  Hexo is running at http://localhost:4000/ . Press Ctrl+C to stop.

前两条表示hexo的启动过程，第三条表示启动结果。

http://localhost:4000/ 是本地博客的启动端口，在浏览器中输入这个地址即可看到你的博客。

Press Ctrl+C to stop 指点击Ctrl+C即可停止hexo运行。

检视完成后停止运行博客。

然后就可以开始编写博文了，毕竟空博客永远只是空博客，要让博客活起来就要学会编写博文。

hexo n "我的第一篇博客文章"

这段命令的作用是创建一个新的博文，其中“我的第一篇博客文章”为博文文件名，可以随意更改。

你所创建的所有博文都储存在 /Blog/source/_posts 中。

你可以用终端进入这个文件夹，然后输入

notepad 我的第一篇博客文章.md

来使用记事本打开博文文件，或者直接在文件管理器中打开博文文件，VSCode也可以直接打开文件。

博文的编辑遵从 markdown 语言，方便快捷，这里不多做赘述。

返回Blog根目录，输入以下命令来清除缓存，更新博客。

hexo clean
hexo g

至此，博客的创建、检阅与编辑部分结束。

第三节-部署博客

博客制作完毕需要将其部署至远端，才能让别人在自己的设备上看到你的博客。

这里介绍如何将自己的博客部署至 Github 。

首先需要先在 Github 上注册一个自己的账号， Github 官网打不开的需要进行代理，这里推荐使用 Fastgithub 虽然但是 fasthub 需要在github上下载。这时可以下载 steam++ 使用里面的 Github 加速功能，来下载 fastgithub ，再用 fastgithub 来代理 Github。

总之需要注册一个 Github 账号。

然后新建一个仓库（repository）

Repository name 这一栏填写 “xxxxxxxx.github.io” ”xxxxxxxx“写自己的Github的名字，例如我的 “Dwmtcat.github.io“

下面的Description（说明）可填可不填，然后点击Create repository创建仓库

然后回到终端里,用以下指令在blog目录下安装git部署插件.

cnpm install --save hexo-deployer-git

如果没有红字的 Error 只有黄字或者红字的 Worning 则并无大碍

然后打开根目录里的 _config.yml 文件 可以用notepad打开 命令与上文一致

将 _config.yml 文件的末尾：

deploy:
type:

改为：

deploy:
type: git
repo: https://github.com/xxxxxxxx/xxxxxxxx.github.io.git
branch: master

然后在终端中输入以下命令来完成git的认证

git config --global user.email "xxx"
git config --global user.name "xxx"

第一个的xxx要写自己注册Github的邮箱，第二个xxx要写自己github的用户名。（虽然随便写也是没有问题的，但是规规整整写比较方便。）

最后输入以下命令将自己的博客上传

hexo d

然后在 Https：//xxxxxxxx.github.io 这个网址上就可以看到自己的博客啦

第四节-博客的美化

你可以在hexo官网上的主题里找自己喜欢的主题

这里选用Butterfly这个主题来作为实例

在终端中输入

git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

来克隆这个主题到自己的博客上

在根目录的 _config.yml 中将：

theme: landscape

改为：

theme: butterfly

然后 清除缓存 更新博客 上传博客三件套。

hexo clean
hexo g
hexo d

之后就可以看到改过主题的博客啦。

日后再次更改博客，完成后也可以通过三件套上传覆盖博客进行博客的更新。

附录

在创建博客时有概率遇到证书问题

clone、pull或push等证书异常
— fatal: unable to access ‘https://github.com/xxx.git/'
— SSL certificate problem: unable to get local issuer certificate

只需要运行这条命令即可

git config --global http.sslverify false
